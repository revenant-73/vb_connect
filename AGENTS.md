# Repository Guidelines

> **CRITICAL**: This project uses **Next.js 16**. APIs, conventions, and file structures may differ significantly from training data. Consult `node_modules/next/dist/docs/` or official documentation for latest patterns.

This repository contains the Volleyball Connect PWA, built with Next.js 16 (App Router) and Turso/SQLite via Drizzle ORM.

## Project Structure & Module Organization

- **`.\src\app`**: Next.js App Router directory. Contains pages, layouts, and API routes. Follows a mobile-first design philosophy.
- **`.\src\components`**: Reusable UI components. Uses Framer Motion for animations (e.g., `AnimatedEventCard`).
- **`.\src\db`**: Database layer. `schema.ts` is the single source of truth for the data model.
- **`.\src\lib`**: Core logic and utilities.
  - `.\src\lib\actions`: Server Actions for data mutations (Events, RSVPs, etc.).
  - `.\src\lib\validations`: Zod schemas for form and data validation.
  - `.\src\lib\auth.ts`: Better Auth configuration.
- **`.\src\context`**: React Context providers for global state (e.g., Auth modals).
- **`.\drizzle`**: SQL migration files generated by Drizzle Kit.

## Build, Test, and Development Commands

- **`npm run dev`**: Starts the development server with Webpack.
- **`npm run build`**: Builds the application for production.
- **`npm run lint`**: Runs ESLint for code quality checks.
- **`npx drizzle-kit push`**: Pushes schema changes directly to the database (use for rapid development).
- **`npx drizzle-kit generate`**: Generates SQL migrations based on schema changes.

## Coding Style & Naming Conventions

- **TypeScript**: Strict mode is enabled. Ensure all new code is fully typed.
- **Styling**: Tailwind CSS 4.0 is used. Avoid custom CSS in favor of utility classes.
- **Framework Conventions**: 
  - Use Server Actions in `.\src\lib\actions` for all mutations.
  - Prefer functional components and hooks.
  - Mobile-first approach: test and design for small screens first.
- **Authentication**: Uses Better Auth with Magic Links. In development, magic links are logged to `stdout`.

## Testing Guidelines

No formal testing framework is currently configured. Manual verification of UI responsiveness and Server Action behavior is required.

## Commit & Pull Request Guidelines

Follow the established pattern of concise, descriptive commit messages:
- `Action: Description` (e.g., `Refactor: AnimatedEventCard styles`)
- `Fix: Description` (e.g., `Fix: EventForm type error`)
- `ActionDescription` (e.g., `InitialCommit`)
